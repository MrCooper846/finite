# docs/PRODUCT_BRIEF.md

# Product Brief

## Product Name

Finite

## Tagline

Scroll like you mean it.

## One-Sentence Description

Finite helps users keep up with selected creators intentionally by creating a finite catch-up queue that opens real platform profiles and ends when the queue is complete.

## Product Thesis

Social media does not need to be infinite.

Many users do not want to delete social media completely. They still care about specific creators, friends, channels, accounts, communities, or topics. The problem is that checking one thing often turns into accidental scrolling because modern social apps are built around infinite feeds, recommendations, and algorithmic discovery.

Finite gives users a deliberate route into social platforms and a clear route back out.

## Core Promise

Keep up with who you care about. Then leave.

## Problem

Users often open social platforms with a clear intention, such as checking a creator, watching a specific channel, or catching up with a friend.

Once inside the platform, they are exposed to:

* infinite scroll
* algorithmic recommendations
* For You pages
* Explore pages
* Reels/Shorts-style feeds
* notification loops
* suggested accounts
* trending content
* engagement-maximising design patterns

This causes users to spend more time than intended and consume content they did not actively choose.

## Target Users

Finite is for people who want to stay connected without being pulled into infinite feeds.

Primary target users:

* people trying to reduce doomscrolling
* digital minimalists
* dumbphone or secondary-phone users
* students and young adults who feel overstimulated by social apps
* ADHD or distraction-prone users
* people who want to support creators without losing control of their time
* people who keep an old smartphone as a deliberate-use media or admin device

## User Jobs

Users want to:

* keep up with specific creators
* avoid accidental scrolling
* reduce wasted time
* make social media feel intentional again
* support creators in the original apps
* avoid deleting every app completely
* use social media in short, bounded sessions

## Main User Story

As a user, I want to create a list of creators I care about, open them one at a time in their original apps, and mark them as checked, so that I can keep up without falling into infinite scroll.

## Core Flow

```text
1. User adds creators manually.
2. User opens Finite.
3. User sees a finite queue.
4. User starts catch-up mode.
5. Finite shows one creator at a time.
6. User opens that creator in the original platform.
7. User returns to Finite.
8. User marks the creator done or skipped.
9. Finite moves to the next creator.
10. When the queue is complete, Finite tells the user they are caught up.
```

## Product Positioning

Finite is not a social media replacement.

Finite is a routing layer for intentional social media use.

It sits before the social app and asks:

```text
Who did you come here to check?
```

Then, when the user has checked them, it says:

```text
You’re done. Leave.
```

## What Finite Is

Finite is:

* a catch-up queue
* a deliberate-use tool
* a social media router
* a behavioural guardrail
* a lightweight companion for existing social apps
* a local-first productivity/lifestyle app

## What Finite Is Not

Finite is not:

* a feed
* a social network
* a recommendation engine
* a creator discovery platform
* an Instagram/TikTok/YouTube replacement
* a scraping tool
* a parental control app
* a blocker-first productivity app
* a shame-based digital wellbeing app

## Product Principles

### 1. The user chooses

Finite should only show creators/accounts the user intentionally added.

### 2. The queue ends

Every catch-up session should have a clear end.

### 3. The original platforms still receive engagement

Finite should open creators in their real platform/app where possible.

### 4. The app should be faster than the bad habit

Opening Finite and starting a catch-up session should feel quicker than opening a social app and navigating manually.

### 5. Completion is the reward

Finite should reward finishing and leaving, not staying.

## v0.1 Product Goal

Build a working local prototype of the core catch-up flow.

The prototype is successful if a user can:

* add several creators
* start a catch-up session
* open each creator externally
* mark creators done or skipped
* finish the queue
* understand the purpose of the app without explanation

## v1.0 Product Goal

Ship a polished App Store-ready version of the local-first app.

The 1.0 version should feel complete even without a backend or platform APIs.

It should include:

* polished onboarding
* creator management
* catch-up queue
* external profile opening
* mark done / skip
* last checked state
* basic reminders
* import/export backup
* basic stats
* privacy-first local data
* a clear completion screen

## Possible Future Features

These are not required for v0.1:

* optional account sync
* YouTube/RSS smart updates
* multiple lists
* custom schedules
* reminders
* export/import JSON
* widgets
* Android release
* one-time Pro/supporter purchase
* browser extension
* platform-specific deep link improvements

## Monetisation Philosophy

Finite should not monetise attention.

Avoid:

* ads
* sponsored creators
* recommendation placements
* engagement-maximising loops
* dark patterns
* artificial usage streaks that encourage opening the app unnecessarily

Possible ethical monetisation later:

* generous free version
* optional one-time Pro purchase
* optional supporter purchase
* paid sync only if there are real server costs
* cosmetic supporter features
* advanced organisation features for power users

## Success Criteria

Finite is successful if users say:

* “I checked what I wanted and left.”
* “This stopped me opening Explore.”
* “I still support the creators I care about.”
* “It made social media feel intentional again.”
* “I use my phone less because of this.”
